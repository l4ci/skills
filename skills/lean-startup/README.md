# lean-startup

Tests a startup, product, or business idea under uncertainty using Eric Ries's Lean Startup method (2011). Instead of writing a big plan and executing it, the venture is treated as a set of untested hypotheses, and the smallest possible experiment is run against the riskiest one. Progress is measured as validated learning about customers, not features shipped, and each cycle ends in a deliberate decision to pivot or persevere.

This skill runs one Build-Measure-Learn loop end to end. It helps surface the leap-of-faith assumptions (including the value and growth hypotheses) as falsifiable statements, then ranks them by risk using parallel analysts working different lenses (most uncertain, most fatal if false, what a skeptical investor would attack), so the first experiment tests the riskiest assumption rather than the easiest. It then designs the minimum viable test with an actionable metric and a pass-and-fail threshold committed in advance, captures what customers actually did, and recommends pivot or persevere on that evidence. The output is a saved markdown experiment document with a loop log.

It guards against the method's common failure modes: the MVP as a small product rather than a learning tool, vanity metrics that rise but test nothing, and results rationalized after the fact because no threshold was set beforehand.

## Starting

**You provide:** the product or business idea and who it is for; optionally the stage, resources, what has been built or tried, and any evidence so far. Something like "help me validate my idea".
