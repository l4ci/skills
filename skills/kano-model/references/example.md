# Example: kano-model

**You invoke it with:**
> "Run a Kano analysis on these features for our solo-freelancer plan"

**It needs from you:**
- SEGMENT (required): solo freelancers on our entry-tier invoicing plan
- FEATURES (required): recurring invoices, automatic late-payment reminders, multi-currency support, a built-in time tracker, AI-drafted invoice descriptions, white-label branding
- CONTEXT (optional): a $12/mo SaaS invoicing tool competing with FreshBooks and Wave; the decision is what to put in the next two quarters of roadmap

**You get back:** a saved markdown report (`kano-model-<subject-slug>-<date>.md`) with a verdict leading on any missing must-be, a classification table grouped by category (must-be / one-dimensional / attractive / indifferent / reverse) with confidence and decay risk, per-feature reasoning, and a recommended feature set and build sequence with a decay timeline.
