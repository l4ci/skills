# scqa-pyramid

Structures a recommendation, set of findings, or decision so the reader gets the answer first and the rest of the document defends it. It uses two linked tools from Barbara Minto's *The Pyramid Principle*, the communication method that became standard at McKinsey. SCQA frames the reader's context in four moves (Situation, Complication, Question, Answer) and delivers the governing thought up front. The pyramid is the hierarchy of supporting ideas beneath that answer, where each level summarizes the level below it. The core idea is answer first: state the conclusion, then let the structure defend it, rather than making the reader assemble it from clues.

The framing fixed in the introduction decides what the whole pyramid must prove, so this skill does not commit to the first framing. Two or three subagents propose competing SCQA framings in parallel, each emphasizing a different complication or reader, which surfaces a different question and answer. A selection pass picks or merges the sharpest, the one whose Question the audience actually has and whose Answer the evidence can support. Then one pass builds the pyramid: the answer on top, a MECE key line of supporting arguments beneath it (deductive or inductive, stated), and the evidence under each. Then two checkers audit it in parallel, one for whether each level summarizes rather than labels and each grouping is one kind, the other for whether the key line is MECE and answers the question the answer raises. The pyramid is then refined at the level where each problem originates.

This skill structures and communicates the message; it does not generate the analysis or make a weak case strong. If the evidence cannot support the answer, the fix is more thinking, not a better outline. It pairs downstream of the analysis skills in this collection, the step where the result of the thinking becomes something a reader can follow, and it shares its Minto lineage with [mece-decomposition](../mece-decomposition/): a pyramid's key line must itself be MECE.
