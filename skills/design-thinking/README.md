# design-thinking

Facilitates a human-centered innovation process through the five Design Thinking modes (Stanford d.school and IDEO framing): Empathize, Define, Ideate, Prototype, Test. The method runs as a loop rather than a fixed sequence. Insight from a test can send the work back to reframing the problem or back to talking to users, and that looping is how the method is meant to run.

This skill guides the work one mode at a time and keeps a living design document. It leads with real user needs (the main way design work drifts is toward the solution and away from the user), and during Ideate it diverges hard before converging: several idea-generator subagents run in parallel, each using a different ideation lens, so the pooled ideas are genuinely varied rather than ten versions of one before the user converges on a few to prototype. Prototypes are kept deliberately cheap and rough, and testing is framed to seek disconfirming feedback rather than applause.

It works alongside other methods: a problem-framing skill can sharpen the Define mode, and a Design Thinking concept can feed a Lean Startup experiment for viability.

**You provide:** the design challenge and who it's for; any prior research or constraints are optional. Something like "facilitate a design thinking process for our patient discharge experience".
